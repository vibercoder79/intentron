# BOO-148 — Projekt-Typ-Marker in project-CLAUDE.md (FEATURE-3)

## Summary

Eine frisch gebootstrappte `project-CLAUDE.md` sagt nicht, ob das Repo ein **aktives** Code-/Deployment-Projekt ist oder eine reine **Governance-Referenz** (nur Docs/Specs, kein Coding). Folge: Agenten behandeln Governance-Repos wie Code-Repos (CI-Erwartung, Implement-Flow) — Fehlannahmen. Fix: das `project-CLAUDE.md`-Template bekommt `{{PROJECT_TYPE_MARKER}}` als erste Zeile nach dem H1; `bootstrap` §4.3a entscheidet beim Setup zwischen AKTIV (Default) und GOVERNANCE-REFERENZ und rendert den passenden Markertext. `migrate_boo_148` rüstet Default AKTIV in bestehende `CLAUDE.md` nach.

## Why

Gefunden beim CI-Post-Mortem von `bko-widerspruch-assistent` (2026-06-03) — FEATURE-3: ohne expliziten Projekt-Typ läuft ein Governance-/Doku-Repo durch denselben Code-/CI-Flow wie ein aktives Code-Projekt. Der Marker macht die Erwartung an das Repo von der ersten Zeile an eindeutig. Linear: https://linear.app/owlist/issue/BOO-148

## What

- `bootstrap/references/file-templates.md` + `.en.md`: `project-CLAUDE.md`-Template bekommt `{{PROJECT_TYPE_MARKER}}` als erste Zeile nach dem H1.
- `bootstrap/SKILL.md` + `.en.md`, §4.3a: Entscheidungsregel AKTIV (Default — Code + Deployment im Repo) vs GOVERNANCE-REFERENZ (nur Docs/Specs, kein Coding). Markertexte:
  - DE: `> **PROJEKT-TYP: AKTIV** — Code + Deployment in diesem Repo` / `> **PROJEKT-TYP: GOVERNANCE-REFERENZ** — nur Docs/Specs, kein Coding`
  - EN: `> **PROJECT TYPE: ACTIVE** — code + deployment in this repo` / `> **PROJECT TYPE: GOVERNANCE REFERENCE** — docs/specs only, no coding`
- `bootstrap/scripts/migrate-to-v2.sh`: neue idempotente `migrate_boo_148()` — rüstet den Default-AKTIV-Marker in bestehende `CLAUDE.md` nach (mit Backup); in `ALL_ISSUES` + `migrate_all()` registriert.
- `bootstrap` SKILL-Version 3.37.0 → **3.38.0**.

## Acceptance Criteria

1. `project-CLAUDE.md`-Template (DE+EN) trägt `{{PROJECT_TYPE_MARKER}}` als erste Zeile nach dem H1. ✓
2. `bootstrap` §4.3a (DE+EN) beschreibt die Entscheidung AKTIV (Default) vs GOVERNANCE-REFERENZ mit den Markertexten. ✓
3. `migrate_boo_148()` rüstet Default AKTIV idempotent nach (Skip wenn Marker vorhanden), funktional getestet. ✓
4. In `ALL_ISSUES` + `migrate_all()` registriert. ✓
5. SKILL-Version auf 3.38.0 angehoben (DE+EN). ✓

## Abgrenzung

Der Marker ändert keinen CI-Flow automatisch — er macht den Projekt-Typ explizit, damit Agenten und Operator die Erwartung an das Repo kennen. Default beim Bootstrap und beim Retrofit ist AKTIV; GOVERNANCE-REFERENZ wird beim Setup bewusst gewählt. Reiner Template-/Skill-Fix, kein Runtime-Code.
