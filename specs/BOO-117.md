# BOO-117 — Bootstrap A.2: Bestands-Quellen-Import (Intent / Repo / Doku als Input)

## Summary
Block A bekommt einen optionalen Schritt **A.2b**, der abfragt, ob bereits ein Intent, ein
GitHub-Repo/Verzeichnis oder andere Doku existiert, und diese Quelle **inhaltlich einliest**, um
`PROJECT_DESC` (und optional `PROJECT_NAME` / einen `stack_hint`) **vorzuschlagen** — statt alles
manuell abzufragen. Sauber optional, kein Bruch ohne Quelle.

## Why
Im BKO-Probelauf (2026-06-02, Probelauf-Punkt #1) musste der Operator die Projektbeschreibung manuell
eintippen, obwohl ein Intent/Repo bereits alle Infos enthielt. A.2 (`SKILL.md:121-127`) fragte nur
Name/Beschreibung/Version; ein bestehendes Repo wurde erst in Block B erfasst — rein als Infrastruktur
(Remote/Merge), **nicht inhaltlich**. Der `intent`-Skill war nirgends als Input-Quelle verdrahtet.

## What
- Neuer optionaler Schritt **A.2b** in `bootstrap/SKILL.md` (DE) + `SKILL.en.md` (EN), direkt nach A.2:
  Frage „Gibt es bereits eine Quelle? (a) Intent-Datei (b) Repo/Verzeichnis (c) andere Doku (d) nein".
- Bei a/b/c: Quelle lesen → `PROJECT_DESC` (+ optional `PROJECT_NAME`, `stack_hint`, Add-ons) **ableiten
  und vorschlagen**; Operator bestaetigt/ueberschreibt. Variable `SOURCE_IMPORT = {type, ref, derived}`.
- Wiederverwendung: Wurde dieselbe Quelle bereits in A.1 (Guided-Discovery, BOO-127) analysiert, wird sie
  hier wiederverwendet — kein Doppel-Einlesen/-Fragen.
- `references/info-gathering.md` + `.en.md`: Hinweis-Callout zum optionalen `SOURCE_IMPORT` ergaenzt.

## Acceptance Criteria
1. In Block A optionale Frage nach vorhandener Quelle (Intent/Repo/Doku). ✓ (§A.2b)
2. Bei Ja: Quelle einlesen, `PROJECT_DESC` (+ ggf. weitere Variablen) vorschlagen, Operator bestaetigt. ✓
3. Sauber optional — kein Zwang, kein Bruch ohne Quelle (Default `d`, `SOURCE_IMPORT.type = none`). ✓

## Result
Umgesetzt im Branch `feat/boo-bootstrap-flow-hardening`. Bewusst als Fundament fuer BOO-127 gebaut
(gemeinsame Quellen-Import-Mechanik). Dateien: `bootstrap/SKILL.md` §A.2b, `bootstrap/SKILL.en.md` §A.2b,
`bootstrap/references/info-gathering.md` + `.en.md`. Leichtgewicht: keine neuen Scripts, reine
Skill-Instruktion.
