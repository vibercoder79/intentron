# BOO-114 — Pre-Flight-Gate in Phase 0 (Voraussetzungs-Check mit Abbruch)

## Summary
Phase 0 bekommt vor jedem Scaffold ein **Pre-Flight-Gate**: Kern-Voraussetzungen (Vorab-Bogen
durchlaufen, Toolchain inkl. `gh`/Image, API-Keys/Zugaenge, Ziel-Verzeichnis/Repo) werden abgefragt;
bei `nein/unklar` **kontrollierter Abbruch ohne Scaffold** mit Deeplinks auf `bootstrap-prep.md`,
HANDBUCH §3 + Anhang A und (scaffold-only) Anhang Y.2 / Container-Profil.

## Why
Probelauf-Punkt #5 (groesster Hebel/kleinster Aufwand): Phase 0 (`SKILL.md:35-51`) fragte nur
„Bereit? [ja/spaeter]", pruefte **keine** Voraussetzungen und brach **nie** ab. Vorhandenes Material
(`docs/onboarding/bootstrap-prep.md`, HANDBUCH Anhang A) war **nicht in den Flow verdrahtet** —
`SKILL.md` referenzierte `bootstrap-prep.md` nirgends. Die einzige maschinelle Pruefung
(`verify-setup.sh`) lief erst am Ende (Phase 7.3b), kein Hard-Abbruch.

## What
- Phase 0 in `0.1 Briefing` + `0.2 Pre-Flight-Gate` gegliedert (`bootstrap/SKILL.md` + `.en.md`).
- Pre-Flight fragt 4 Kern-Bloecke ab (Vorab-Bogen / Toolchain inkl. `gh`+Image / Keys+Zugaenge /
  Verzeichnis+Repo); scaffold-only explizit kommuniziert (Briefing + Pre-Flight + Abbruch).
- Abbruch-Block mit Deeplinks: `bootstrap-prep.md`, HANDBUCH §3 + Anhang A, Anhang Y.2 / Container-Profil
  (BOO-81), Verweis auf gefuehrte Install-Deeplinks (BOO-115). „Starte /bootstrap erneut" statt Scaffold.

## Acceptance Criteria
1. Phase-0-Pre-Flight-Block fragt Kern-Voraussetzungen ab (Keys, Zugaenge, Tools/Image, Checkliste). ✓
2. Bei nein/unklar → kontrollierter Abbruch mit Hinweis auf `bootstrap-prep.md` + HANDBUCH Anhang A (DE/EN). ✓
3. „scaffold-only"-Charakter kommuniziert (verzahnt mit BOO-115). ✓

## Result
Umgesetzt im Branch `feat/boo-bootstrap-flow-hardening`. `gh` ist im Pre-Flight-Tool-Block bereits
genannt (forward-konsistent zu BOO-123, das `gh` in Anhang A + §3 + bootstrap-prep verankert). Reine
Skill-Instruktion, keine neuen Scripts (Leichtgewicht).
