# BOO-141 — ESLint-Config: React-/Browser-Globals für TSX-Projekte

## Summary

Das generierte `eslint.config.mjs` aktiviert `js.configs.recommended` (inkl. `no-undef`), liefert für den TypeScript/TSX-Pfad bisher aber **nur** den Hinweis „files-Glob auf `.ts/.tsx` ausweiten" — **ohne** React- und Browser-Globals. Folge: jede `.tsx`-Datei mit JSX wirft `'React' is not defined` (`no-undef`), und `window`/`document`/`fetch` sind ebenfalls undefiniert. Fix: ein dedizierter Frontend/React-Block mit `...globals.browser` **und** `React: 'readonly'` (Paket `globals`), gerendert für Stacks b/c mit React/JSX.

## Why

Gefunden beim ersten CI-Lauf von `bko-widerspruch-assistent` (Next.js 16, 2026-06-03): `'React' is not defined  no-undef` bei jeder TSX-Datei. Der Config-Block im Repo-SSoT ist Node/CommonJS-zentriert; der Frontend-Fall war nur als Prosa-Hinweis angedeutet, nicht als renderbarer Block. Linear: https://linear.app/owlist/issue/BOO-141

## What

- `bootstrap/references/file-templates.md` + `.en.md`, §`eslint.config.mjs`: Abschnitt „Mit React / Frontend (TSX)" erweitert — `npm install --save-dev globals`, `import globals from 'globals';` und ein Frontend-Block (`files: ['**/*.ts','**/*.tsx']`, `globals: { ...globals.browser, React: 'readonly' }`, `no-undef: error`).
- `bootstrap/SKILL.md` + `.en.md`, Phase 4.4: Render-Regel „React/Frontend mit JSX → Frontend-Globals-Block (Pflicht)".
- `bootstrap/scripts/migrate-to-v2.sh`: `migrate_boo_141()` — Manual-Hinweis-Stil (Config ist operator-gerendert, analog `migrate_boo_2`), nur bei erkanntem React/TSX und fehlenden browser-Globals; in `ALL_ISSUES` + `migrate_all()`.

## Acceptance Criteria

1. Frontend/React-Block in `file-templates.md` **und** `.en.md` vorhanden (DE/EN-Parität für den Frontend-Fall). ✓
2. SKILL.md (DE+EN) nennt den Frontend-Globals-Block als Pflicht bei Stack b/c mit JSX. ✓
3. `migrate_boo_141()` idempotent + React-gegated (kein Rauschen bei Node-Backend). ✓
4. Nur Template-/Config-Dateien — kein Runtime-Code. ✓

## Hinweis (entdeckte Drift)

DE- und EN-Basis-Config-Block (`eslint.config.mjs`) divergieren vor-bestehend (DE: `files`-Block + commonjs + Node-Globals; EN: kein `files`-Block + `module`). BOO-141 vereinheitlicht den **Frontend-Block** in beiden; die vollständige Angleichung des Basis-Blocks ist separat (nicht Scope dieser Story).
