# BOO-143 — Perf-Gate skippt bei leerer Baseline (Option A)

## Summary

Der generierte `.github/workflows/perf.yml` failt heute bewusst (`exit 1`), wenn die Performance-Baseline für einen Service fehlt — und ein frisches Projekt hat `journal/perf-baseline.json` mit `services: []`. Der erste CI-Lauf bricht also immer ab (zusätzlich verschärft durch den `autocannon`-devDep, der nur als Doku-Hinweis existiert, und den `exit 1`-Platzhalter im Service-Start). Fix (Option A): ein `Check prerequisites`-Step liest die Baseline; ist sie leer/fehlend → Benchmarks **skippen**, Gate bleibt **grün** (`exit 0`). Sobald die Baseline befüllt ist, läuft das Gate normal.

## Why

Gefunden beim ersten CI-Lauf von `bko-widerspruch-assistent` (2026-06-03): `Error: Cannot find module 'autocannon'` bzw. Abbruch am Perf-Gate. Operator-Entscheidung: Option A (sauber skippen) statt Option B (autocannon-devDep + Stub-Baseline) — Code-Crash-Leichtgewicht-Prinzip, kein Drift-Risiko durch Stub-Werte. Reines Frontend (Stack b) bekommt ohnehin Lighthouse statt perf.yml; Bug trifft Node-Backend/Full-Stack. Linear: https://linear.app/owlist/issue/BOO-143

## What

- `bootstrap/references/file-templates.md` + `.en.md`: perf.yml-Block bekommt nach `Checkout` einen `Check prerequisites`-Step (`id: prereq`, liest `services.length`; leer/fehlend → `skip=true`); die Steps Start service / Wait / Run bench (Node) / Run bench (Python) / Compare against baseline werden mit `if: steps.prereq.outputs.skip == 'false'` gegated. Doku-Note „Erstbefüllung" umgeschrieben (skippt grün statt failt).
- `bootstrap/scripts/migrate-to-v2.sh`: gleicher Skip-Step im `migrate_boo_16()`-Heredoc; neue `migrate_boo_143()` — detektiert fehlenden Skip in Bestands-`perf.yml` und gibt Warn + Manual-Hinweis (sauberster Pfad: `migrate_boo_16 --force`, da perf.yml operator-spezifische Service-Matrix trägt — kein riskanter Auto-Patch der `if`-Guards); in `ALL_ISSUES` + `migrate_all()`.

## Acceptance Criteria

1. perf.yml (DE+EN+Heredoc) hat den `Check prerequisites`-Step; 5 nachgelagerte Steps sind skip-gegated. ✓
2. Frisches Projekt (`services: []`): Perf-Gate läuft grün durch (skippt). ✓
3. Befüllte Baseline: Gate vergleicht normal (PASS/WARN/FAIL unverändert). ✓
4. `migrate_boo_143()` idempotent (Skip wenn bereits vorhanden), funktional getestet. ✓
5. Option A bewusst minimal — kein `autocannon`-devDep, keine Stub-Baseline. ✓
6. Nur Template-/CI-Dateien — kein Runtime-Code. ✓
