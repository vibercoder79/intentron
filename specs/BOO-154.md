# BOO-154 — ideation/implement: Konvention gegen Cross-Session-Drift

## Summary
Kleine Konventions-Ergänzung in `ideation` + `implement`, damit das **ausgelieferte** Framework Cross-Session-Drift (Nummern-/ID-Kollisionen bei parallelen Sessions/Entwicklern) explizit vermeidet. Folge aus BOO-153: Der Wave-Guard bleibt repo-intern, aber die generelle Lektion gehört als Konvention in die Kunden-Skills.

## Why
In Kundenprojekten kommen Story-Nummern vom Backlog-Tool (zentral), aber Spec-Dateinamen werden manuell vergeben — bei parallelem Arbeiten droht derselbe Drift (Nummern-Kollision + Repo↔Backlog-Versatz), der intentron-intern in BOO-153 auftrat. Die Konvention macht die Vermeidung explizit.

## What
- **`ideation/SKILL.md` (+ `.en.md`):** Schritt 6 — Hinweis „Backlog-first": Nummer **vom Backlog-Tool** holen, dann `specs/<PREFIX>XXX.md` mit genau dieser Nummer benennen; nie manuell raten/parallel vergeben; vor Vergabe gegen das Tool prüfen. `ideation` 2.7.0 → 2.8.0 (SKILL+README DE+EN).
- **`implement/SKILL.md` (+ `.en.md`):** Schritt 0c — Cross-Session-Hinweis: Pre-Flight isoliert Agenten EINER Story (Ebene 3); mehrere Menschen/Sessions → eigener Klon / `git worktree` pro Session (Ebene 1/2). `implement` 2.12.0 → 2.13.0 (SKILL+README DE+EN).
- Beide verweisen auf `docs/kollisionsschutz-drei-ebenen.md`.
