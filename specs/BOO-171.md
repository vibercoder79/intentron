# BOO-171 — implement: mechanische Lint-Loops auf Haiku-Subagent delegieren (BOO-170-Folge)

## Summary
Schließt die von BOO-170 offengelassene interne Tier-Lücke — schlanke Variante: die mechanischen Iterations-Loops von `/implement` (6a ESLint/Ruff, 6a-bis Semgrep) werden an einen **`lint-fixer`-Subagenten mit `model: haiku`** delegiert, während Code-Kern (Schritt 5) und Security-Findings (6e) im Opus-Parent bleiben. Kostenhebel ohne Qualitätsverlust am Code.

## Why
BOO-170 startet `/implement` im Daemon mit **einem** Top-Level-Modell (opus) — damit liefen auch die stumpfen „fix bis grün"-Loops auf opus (~19× teurer im Output als haiku, ohne Reasoning-Nutzen). Operator-Entscheidung: Loops auf haiku, Code+Security opus.

## Spike-Ergebnis (verifiziert)
- **Mechanik:** Subagenten tragen `model:` im Frontmatter; ein opus-Parent kann einen haiku-Child spawnen (Auflösung env > per-call > Frontmatter > Parent). Zuverlässig = **benannter** Subagent (`.claude/agents/*.md`); per-call-Syntax undokumentiert. **Kosten-Attribution pro Subagent ist CC-seitig nicht garantiert** (dokumentierte Unsicherheit).
- **implement-Realität:** Das `meta.json`-Schema trackt bereits `skill_invoked: implement-iterations`, `model_tier: haiku` — aber der 6a-Loop lief **faktisch inline** (keine Delegation). Das Schema war aspirational; diese Story macht die Delegation real.
- **Folge aus BOO-170:** Code-Kern + Security (6e) sind bereits opus → **einziger realer Gewinn = 6a/6a-bis-Loops auf haiku** (ein separater opus-Security-Subagent wurde bewusst weggelassen).

## What
- **Neu `implement/references/lint-fixer.agent.md`** — benannte Subagent-Definition (`model: haiku`, tools Read/Edit/Write/Bash), rein mechanischer Fix-Loop mit harten Grenzen (keine Features/Architektur, Gate-Config nicht „grün tricksen").
- **implement Schritt 6a (DE+EN):** Delegations-Block — 6a/6a-bis an `lint-fixer` (haiku); Code-Kern (5) + Security (6e) bleiben opus; graceful degradation (Subagent aus → Parent iteriert selbst, kein Hard-Block).
- **Version implement 2.14.0 → 2.15.0** (SKILL + README, DE+EN).
- `model-tiers.json` **unverändert** — `implement-iterations: haiku` existiert bereits.
