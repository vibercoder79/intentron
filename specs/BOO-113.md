# BOO-113 — Komplementäres Tooling: setup-checklist als eigenes öffentliches Repo + Verweis

## Summary
Die `setup-checklist` (Best-Practice-Maschinen-Setup fuer Claude Code) wird losgeloest vom Framework als eigenes **oeffentliches** Repo bereitgestellt; intentron verweist im README (DE+EN) + HANDBUCH Anhang Z.4 darauf inkl. der Reihenfolge-Regel. Sync ueber erweitertes `publish_skill.py` (Fan-out), kein Hand-Drift.

## Why
intentron ist **public**, das Skill-Repo `claudecodeskills` ist **private** — ein oeffentlicher intentron-README kann nicht auf einen privaten Skill verweisen. Damit die Checkliste neben dem Framework oeffentlich nutzbar ist, braucht sie ein oeffentliches Zuhause. Standalone (statt vendored) passt zur Positionierung: die Checkliste laeuft **vor und unabhaengig** von jedem Projekt (Maschinen-Ebene), bootstrap macht die **Projekt**-Governance.

## What
- **Neues oeffentliches Repo** `vibercoder79/claude-code-setup-checklist` (MIT), Skill im Repo-Root, DE+EN.
- **Komplementaritaets-Notiz + Reihenfolge-Regel** in der Skill-README-Quelle (DE+EN) — reist bei jedem Sync mit.
- **`publish_skill.py` Fan-out** (`PUBLIC_MIRRORS` + `mirror_to_public_repo`): ein Publish schreibt nach `claudecodeskills` (privat, Source-Mirror) UND ins oeffentliche Repo. Single Source of Truth bleibt `~/.claude/skills/setup-checklist`.
- **intentron README** (DE+EN): neuer Abschnitt „Komplementäres Tooling — Maschinen-Setup" mit Link + Reihenfolge `global → /bootstrap → audit/additiv`.
- **HANDBUCH Anhang Z.4** (DE+EN): derselbe Verweis + Regel.

## Reihenfolge-Regel (dokumentiert)
1. `setup-checklist global` — Maschinen-Best-Practice (Effort, Sandbox, Permission-Modi, MCP, globale CLAUDE.md).
2. `/bootstrap` — Projekt-Governance; **besitzt** Projekt-`CLAUDE.md`, `CONVENTIONS.md`, Hooks, `environment.json`.
3. optional `setup-checklist audit`/`projekt` **additiv** (`.claudeignore`, `.gitignore`, `CLAUDE.local.md`) — Projekt-`CLAUDE.md`/Guard-Hook NICHT erneut anlegen.

Faustregel: Maschine + Hygiene = Checkliste · Projekt-Governance = bootstrap.

## Acceptance Criteria
1. Oeffentliches Repo existiert (PUBLIC), enthaelt den vollstaendigen Skill (DE+EN) + LICENSE.
2. intentron README (DE+EN) verweist mit Reihenfolge-Regel darauf.
3. HANDBUCH Anhang Z.4 (DE+EN) dokumentiert dasselbe.
4. `publish_skill.py` synct setup-checklist nach claudecodeskills UND ins oeffentliche Repo (idempotent, ueberspringt sauber wenn nicht geklont).

## Result
Umgesetzt, v0.7.9 Wave AS. Repo: https://github.com/vibercoder79/claude-code-setup-checklist
