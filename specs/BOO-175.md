## Summary

Aufklärung + Prävention: **BOO-172 wurde über ZWEI PRs nach `main` gemergt** — #69 („docs: Runbook …
(BOO-172)") und #70 („Feat/boo 172 runbook tmux vps", generischer Branch-Titel **ohne** BOO-Nummer).
`main` ist inhaltlich konsistent (docs-drift grün, keine doppelten Dateien), aber die PR-/Merge-Historie
ist unsauber. Root-Cause dokumentieren, Vollständigkeit von `main` verifizieren, **Guard/Konvention**
gegen Wiederholung etablieren. **Kein** History-Rewrite.

## Why

Zwei Merges für eine Story bedeuten: unklare Historie, doppelte Squash-Commits, und ein PR-Titel ohne
BOO-Nummer kann (Memory: BOO-Nummern Linear-first) beim Merge das falsche/kein Linear-Issue verknüpfen.
Vermutete Ursache: Push auf den Feature-Branch **nach** PR-Erstellung in Kombination mit der Merge-/
Branch-Delete-Reihenfolge in einer Session — genau zu verifizieren.

## What

1. **Read-only-Analyse** von #69 + #70 (`gh pr view 69`, `gh pr view 70`, `git log`): prüfen ob #70
   inhaltlich ein No-op/Duplikat von #69 war (dann nur dokumentieren) oder ob Inhalte nur in einem der
   beiden lagen.
2. **Root-Cause dokumentieren** — wie entstanden zwei PRs/Merges für BOO-172?
3. **Vollständigkeit verifizieren** — `main` enthält alle BOO-172-Inhalte, keine verwaisten/halben
   Commits (kein Inhalt nur in #70).
4. **Guard/Konvention** gegen Wiederholung, z. B.:
   (a) vor `gh pr create` prüfen, ob für den Branch schon ein PR offen ist;
   (b) PR-Titel **muss** BOO-Nummer enthalten (Memory: Linear-first) — als Punkt in die
   Merge-Checkliste / `framework-update`-Runbook bzw. `CONVENTIONS.md` aufnehmen.
5. **Verwaiste Branches** der Doppel-PRs aufräumen (falls vorhanden).

## Acceptance Criteria

1. Root-Cause dokumentiert: wie entstanden zwei PRs/Merges für BOO-172? ☐

3. **Guard/Konvention** gegen Wiederholung: (a) PR-Existenz-Check vor `gh pr create`; (b) PR-Titel muss
   BOO-Nummer enthalten — in Merge-Checkliste/`framework-update`-Runbook bzw. CONVENTIONS aufgenommen. ☐
4. Offene/verwaiste Branches der Doppel-PRs aufgeräumt (falls vorhanden). ☐
