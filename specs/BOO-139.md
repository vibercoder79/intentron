# BOO-139 — Standard-Projektstruktur: journal/daily/ + Session-Ende-Daily-Note-Routine in generierter Projekt-CLAUDE.md

## Summary
Daily Notes docken als `journal/daily/YYYY-MM-DD.md` an das **bestehende** `journal/`-Baseline-Artefakt (BOO-61) an, und die generierte Projekt-`CLAUDE.md` bekommt eine **Session-ENDE-Routine**: am natürlichen Sessionende fragt Claude *„Soll ich die Daily Note schreiben?"* → `journal/daily/YYYY-MM-DD.md` (was gemacht, Entscheidungen, offene Punkte). Macht aus dem `repo-docs`-Projekt ein lebendes, projekt-lokales Logbuch **ohne Obsidian** — das Gegenstück zum Session-START-Loop (BOO-129).

## Why
Spiegelt Tobias' bewährtes Vault-Pattern (Daily Note pro Tag, „Schluss für heute → Daily Note") auf die VPS. Die nächste Session weiß sofort, wo man stehen geblieben ist — starkes, leichtgewichtiges Projektmanagement ohne Zusatz-Tooling.

## Befund (verifiziert 2026-06-03)
- `references/file-templates.md` CLAUDE.md-Template hat die Session-**START**-Routine (BOO-129), aber **keine** Session-ENDE-Routine.
- `journal/` existiert bereits als Baseline-Artefakt (`SKILL.md` BOO-61-Regel) → **kein neuer Top-Level-Ordner**, nur der Subordner `journal/daily/` + Konvention.
- `references/project-documentation-ssot.md` beschreibt den repo-docs-Loop, erwähnt Daily Notes / `journal/daily/` aber nicht.

## What
- **`references/file-templates.md` CLAUDE.md-Template (+ `.en.md`):** Session-START-Routine um „+ letzte `journal/daily/`-Notiz" ergänzen; neue **Session-ENDE-Routine** „Daily Note schreiben?" + Schreib-Konvention `journal/daily/YYYY-MM-DD.md` (was gemacht / Entscheidungen mit Verweis auf `decisions/` / offene Punkte).
- **`references/project-documentation-ssot.md` (+ `.en.md`):** im Leichtgewicht-SecondBrain-Abschnitt `journal/daily/` als Tages-Logbuch nennen (Abgrenzung zu `meetings/`/`decisions/`).
- **HANDBUCH Anhang U Weg 2 (DE+EN):** `journal/daily/` in der Pro-Projekt-Struktur erwähnen.

## Acceptance Criteria
1. CLAUDE.md-Template enthält Session-ENDE-Routine + `journal/daily/`-Schreib-Konvention (DE+EN). ✓
2. Session-START-Routine liest zusätzlich die neueste `journal/daily/`-Notiz (DE+EN). ✓
3. `project-documentation-ssot.md` nennt `journal/daily/` (DE+EN). ✓
4. HANDBUCH Anhang U Weg 2 erwähnt `journal/daily/` (DE+EN). ✓
5. DE+EN-Parität. ✓

## Abgrenzung
Kein Dataview/Auto-Aggregation (Obsidian-only) — bewusst nur Markdown + relative Links, damit die Basis GitHub-/DMS-portabel bleibt. Daily Notes nutzen das bestehende `journal/` (kein Parallel-Ordner). Schwester-Story: BOO-138.
