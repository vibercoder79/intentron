# BOO-142 — Semgrep-Workflow: Container raus + `upload-sarif@v4` in allen drei Workflows

## Summary

Der generierte `.github/workflows/semgrep.yml` lief in einem Docker-Container (`container: returntocorp/semgrep`). Innerhalb eines Containers schlägt `actions/checkout` bei PRs konsistent fehl (`fatal: repository '...' not found`) — der Security-Gate ist auf PRs dauerhaft rot. Fix: Container entfernen, Semgrep per `pip install semgrep` auf `ubuntu-latest`. Zusätzlich: `github/codeql-action/upload-sarif` von **v3 → v4** (v3 deprecated ab Dez 2026) in **allen drei** Workflows (semgrep, eslint, ruff) + Upload-Step mit `if: always() && hashFiles('...sarif') != ''` absichern. Die Manifest-Reader-Logik (SSoT mit Pre-Commit-Hook) bleibt unverändert.

## Why

Gefunden beim ersten CI-Lauf von `bko-widerspruch-assistent` (2026-06-03). `actions/checkout` checkoutet in allen anderen Workflows (ohne Container) problemlos — nur der containerisierte Semgrep-Job scheitert. v3-Deprecation betrifft alle drei SARIF-Uploads, daher Konsistenz-Upgrade. Linear: https://linear.app/owlist/issue/BOO-142

## What

- `bootstrap/references/file-templates.md` + `.en.md`: semgrep.yml-Block ohne `container:`, neuer `- name: Install Semgrep / run: pip install semgrep`-Step nach `checkout`; `upload-sarif@v4` + `hashFiles`-Guard in semgrep/eslint/ruff; Prosa-Stellen `@v3 → @v4`.
- Gleiche Änderungen in den Heredocs `migrate_boo_4()` (semgrep) und `migrate_boo_28()` (eslint/ruff) sowie Prosa in `bootstrap/SKILL.md`/`.en.md`, `HANDBUCH.md`/`.en.md`, `migration-checklist-v1-to-v2.md`/`.en.md`.
- `bootstrap/scripts/migrate-to-v2.sh`: neue idempotente `migrate_boo_142()` — entfernt `container:`-Zeile aus Bestands-`semgrep.yml` (Backup), fügt pip-Step via awk nach checkout ein, hebt `upload-sarif@v3 → v4` in allen Workflows an; in `ALL_ISSUES` + `migrate_all()`.
