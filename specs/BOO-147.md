# BOO-147 — /implement: Remote-CI-Loop nach dem Push (BUG-6)

## Summary

`/implement` validiert heute lokal (Validate-Fix-Learn-Loop) und pusht in Schritt 5 — danach ist Schluss. Ob der **Remote-CI-Lauf** auf GitHub grün wird, prüft der Skill nicht; ein roter CI-Lauf bleibt unbemerkt, bis der Operator manuell nachschaut. Fix: ein neuer Sub-Step **6h Remote-CI-Loop** (DE+EN), der nach dem `git push` aus Schritt 5 läuft: `gh run watch --exit-status` → bei Failure `gh run view --log-failed` → Diagnose/Fix → re-push → max 3 Iterationen → Operator-Eskalation. Remote-Pendant zum lokalen Validate-Fix-Learn-Loop, mit Graceful Degradation.

## Why

Gefunden beim CI-Post-Mortem von `bko-widerspruch-assistent` (2026-06-03) — BUG-6: lokal grün, Remote-CI rot, und niemand merkt es, weil `/implement` nach dem Push abschließt. Der lokale Loop deckt nur die lokale Toolchain ab; GitHub-spezifische Failures (Permissions, Runner-Umgebung, SARIF-Upload) entgehen ihm. Linear: https://linear.app/owlist/issue/BOO-147

## What

- `implement/SKILL.md` + `.en.md`: neuer Sub-Step **6h Remote-CI-Loop** nach dem Push aus Schritt 5. Ablauf: `gh run watch --exit-status` (auf den durch den Push getriggerten Lauf) → bei Failure `gh run view --log-failed` → Diagnose → Fix → re-push → erneut watchen → **max 3 Iterationen**, danach Operator-Eskalation.
- Graceful Degradation: wenn `gh` fehlt / nicht eingeloggt / kein Remote konfiguriert → Skip mit Hinweis, **kein** Hard-Fail.
- `implement` SKILL-Version 2.11.1 → **2.12.0**.

## Acceptance Criteria

1. Sub-Step 6h in `implement/SKILL.md` **und** `.en.md` vorhanden (DE/EN-Parität). ✓
2. Loop läuft nach dem `git push` aus Schritt 5, watcht den getriggerten Lauf via `gh run watch --exit-status`. ✓
3. Bei Failure: `gh run view --log-failed` → Diagnose/Fix → re-push → max 3 Iterationen → Operator-Eskalation. ✓
4. Graceful Degradation bei fehlendem/nicht-eingeloggtem `gh` oder fehlendem Remote (Skip + Hinweis, kein Hard-Fail). ✓
5. SKILL-Version auf 2.12.0 angehoben (DE+EN). ✓

## Abgrenzung

Remote-Pendant zum bestehenden lokalen Validate-Fix-Learn-Loop — ersetzt diesen nicht, sondern ergänzt ihn nach dem Push. Keine Änderung an den lokalen Validate-Steps. Reiner Skill-/Prozess-Fix, kein Template- oder Runtime-Code.
