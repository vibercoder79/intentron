# BOO-127 — Guided Stack-Discovery + Framework/TypeScript-bewusste Stack-Frage

## Summary
Die Stack-Frage A.1 wird **framework-aware** (Meta-Frameworks Next.js/Nuxt/SvelteKit/Remix/Astro als
erkennbare Full-Stack-Variante `c)`) und **TypeScript-first** (`LANG_VARIANT = ts|js`; bei `ts`:
`tsconfig.json` + `typescript-eslint` + `tsc --noEmit`-CI-Gate, `metadata.stack = node-typescript`).
Neuer **Guided-Discovery-Modus** A.1a: bei Unsicherheit leitet der Bootstrap aus einer Quelle (Repo/
Intent/Doku — via A.2b/BOO-117) **oder** einer Vorhaben-Beschreibung einen Stack-**Vorschlag** ab.

## Why
Im BKO-Probelauf wurde ein Next.js/TS/Tailwind-Full-Stack-Repo faelschlich als „a) Node.js Backend"
gebootstrappt → manuelle Umbau-Tabelle noetig. Ursache: A.1 (`SKILL.md:61-78`) war nach Sprache/Schicht
geschnitten, nicht framework-aware; TypeScript war nirgends first-class (alle Wege → `eslint.config.mjs`
JS, kein `tsconfig`/`tsc`-Gate); es gab keinen „beschreibe dein Vorhaben → Stack-Vorschlag"-Pfad. Diese
Story behebt die **Ursache** (richtig waehlen) statt nachtraeglich umzubauen (Leichtgewicht-Prinzip:
kein separater Re-Stack-Pfad fuer den Einzelfall).

## What
- **A.1 framework-aware** (`bootstrap/SKILL.md` + `.en.md`): `a)` Node.js/TS Backend, `c)` Full-Stack
  inkl. Meta-Frameworks; Hinweis „Meta-Framework → c), nicht a)".
- **TypeScript first-class:** Folgefrage `TypeScript oder JavaScript? (Default ts)` → `LANG_VARIANT`.
  Bei `ts`: `tsconfig.json`, `typescript-eslint`, `tsc --noEmit`-CI-Gate, `metadata.stack=node-typescript`.
  (environment.json-Detection existierte bereits — `file-templates.md:1939-2004`; jetzt auch zur Bootstrap-Zeit gescaffoldet.)
- **A.1a Guided Stack-Discovery:** bei `e)`/Unsicherheit → (a) Quelle analysieren (A.2b-Mechanik) ODER
  (b) Vorhaben beschreiben → Stack-Vorschlag; Operator bestaetigt/ueberschreibt; Entscheidung als ADR;
  „noch offen" statt stillem JS-Default.
- **§4.4 Linting** (`SKILL.md` + `.en.md`): TS-Branch (typescript-eslint, tsconfig) + Typecheck-Gate-Callout.
- **Templates** (`file-templates.md` + `.en.md`): neue Sektionen `tsconfig.json (BOO-127)` +
  `tsc --noEmit Typecheck (BOO-127)`.

## Acceptance Criteria
1. Framework-aware Stack-Optionen statt nur Sprache/Schicht. ✓ (A.1, Meta-Frameworks in `c)`)
2. TypeScript first-class: `tsconfig.json`, `typescript-eslint`, `tsc --noEmit`-CI, `stack: node-typescript`. ✓
3. Guided-Discovery-Modus (Quelle analysieren ODER beschreiben → Vorschlag), baut auf BOO-117. ✓ (A.1a)
4. Vorschlag bestaetigt/ueberschrieben + als ADR dokumentiert. ✓
5. DE+EN. ✓

## Result
Umgesetzt im Branch `feat/boo-bootstrap-flow-hardening`. Buchstaben-Mapping `a–e` bewusst erhalten →
keine Regression in den STACK_CHOICE-abhaengigen Folgephasen (Semgrep-Layer-2, CI-Lint, Observability-
Defaults). Guided-Discovery teilt sich die Quellen-Import-Mechanik mit BOO-117 (A.2b).
