# BOO-103 — Audit-/CISO-Perspektive-Runbook

## Summary
Ein Runbook, das aus AUDIT-Perspektive zeigt, wie ein CISO/CIO/CTO verifiziert, dass das Dev-Team die Regeln eingehalten hat. Reiner Aggregator des bereits vorhandenen Materials — keine neue Mechanik. Plus Fix eines toten Verweises.

## Why
Die Audit-Evidenz existiert (Spec-Gate, audit-trace.sh, journal/reports lokal+CI, Branch-Protection, dpo-audit.py, verify-setup, Vier-Augen), war aber ueber 6 Quellen verstreut — keine zusammenhaengende Verifizierer-Sicht. Im Konzern ist die Frage „hat das Team die Regeln eingehalten?" auditrelevant.

## What
- `docs/runbooks/audit-perspective.md` (DE) + `.en.md` (EN): „Audit-Frage → Beleg → Ort"-Tabelle (11 Fragen) + „So pruefst du konkret"-Schritte (Beispiel-Kommandos: audit-trace.sh, gh api branch-protection, gh run, git blame) + Caveats (journal/reports gitignored → CI-Artifacts maßgeblich; Session-Log-Retention; Vier-Augen = Konvention, nicht erzwungen).
- Fix: `bootstrap/scripts/audit-trace.sh` verwies auf nicht existierende `GOVERNANCE.md` → jetzt `docs/runbooks/audit-perspective.md` + CONVENTIONS §Audit-Trail.
