# BOO-124 — Branching-Standard benennen & positionieren (Default Trunk-Based) + Sketch + ADR

## Summary
Den **Default-Branching-Standard** des Frameworks explizit benennen und als verkaufbaren Standard
positionieren: **Trunk-Based mit geschützter `main` + kurzlebige Feature-Branches + PR + Required Checks.**
Plus Verkaufs-Einzeiler, DE/EN-Sketch (Render-Loop) und ADR. Die bestehende Options-Tabelle wird als
„Alternativen" referenziert — nicht dupliziert.

## Why
Probelauf-Folgefrage: „Mit welchem Standard kommt ihr?" Die Branch-Strategie-Optionen (Trunk/Feature/
GitFlow) standen im HANDBUCH (Anhang R) als gleichwertige Trade-off-Tabelle — **ohne benannten Default**,
ohne Sketch, ohne ADR, ohne Verkaufs-Einzeiler.

## What
- **HANDBUCH Anhang R §Layer-1** (DE+EN): benannter Default-Standard-Block + Begründung (leichtgewicht,
  agenten-tauglich, geschützte main = QS) + **Verkaufs-Einzeiler** + Sketch-Embed + ADR-Verweis; die
  bestehende Options-Tabelle als **„Alternativen (wenn nicht unser Default)"** re-gerahmt (nicht dupliziert).
- **ADR** `docs/domain/adrs/branching-standard.md` + `.en.md` (Status accepted, Kontext/Entscheidung/
  Begründung/Alternativen/Konsequenzen).
- **Sketch** `docs/assets/boo-124-branching-standard.excalidraw` + `.png` (DE) + `.en.*` — OWLIST-Farben,
  Render-Loop (write→render→Read→fix; Iter 1 ASCII-Umlaut-Defekt → Iter 2 echte Umlaute, Read-verifiziert).

## Acceptance Criteria
1. Default-Standard explizit benannt (Trunk-Based + geschützte main + PR + Required Checks) + Begründung. ✓
2. Verkaufs-Einzeiler aufgenommen. ✓ (HANDBUCH + ADR, DE+EN)
3. DE/EN-Excalidraw-Sketch (Render-Loop, OWLIST-Farben). ✓ (Read-verifiziert)
4. Als ADR festgehalten (`docs/domain/adrs/`). ✓ (DE+EN)
5. Alternativen-Anhang: Options-Tabelle als „wenn nicht Default" referenziert, nicht dupliziert. ✓
6. DE+EN-Parität. ✓

## Anti-Fabrikation
Story nannte „ADR-Heimat existiert: docs/domain/adrs/ + ARCHITECTURE_DESIGN.md". Diese existieren nur als
**generierte Projekt-Pfade**, nicht im Framework-Repo. Daher `docs/domain/adrs/` im Framework-Repo **neu
angelegt**; der `ARCHITECTURE_DESIGN.md §9`-Verweis ist als gebootstrappte-Projekt-Konvention dokumentiert.

## Result
Umgesetzt im Branch `feat/boo-stack-mcp-branching`. Sketch DE+EN gerendert + via Read-Tool geprüft.
