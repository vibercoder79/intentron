# BOO-119 — SonarQube/SonarCloud-Setup-Runbook (zwei Szenarien) + Bootstrap-Deeplink im Provider-Postflight

## Summary
Das externe SonarCloud-Setup als DE+EN-Runbook mit **zwei Szenarien** dokumentieren (stabiler Anker
**HANDBUCH Anhang AA**) und aus Bootstrap **D.5** (bei „ja") + **Provider-Postflight (BOO-58)** darauf
deeplinken — gleiche Philosophie wie BOO-115 (ins HANDBUCH, Bootstrap verlinkt).

## Why
Probelauf-Folgefrage: D.5 generiert `sonar.yml`, aber die **SonarCloud-seitige** Einrichtung (Account,
„Import from GitHub", Token, `SONAR_TOKEN`-Secret, `properties` abgleichen) war nicht als DE+EN-Anleitung
dokumentiert — nur abstrakte `[MANUAL]`-Stichpunkte. D.5 deeplinkte auf kein Runbook.

## What
- **HANDBUCH Anhang AA** (DE+EN, neuer stabiler Anker — A–Z waren erschöpft): zwei Szenarien
  (A: Account existiert / B: von 0) mit konkreten Schritten + `gh secret set SONAR_TOKEN` +
  `sonar.organization`/`sonar.projectKey`-Abgleich + „erster `git push` triggert `sonar.yml`" +
  Connected-Mode-Hinweis. Verzahnt mit BOO-122.
- **Bootstrap D.5** (`SKILL.md` + `.en.md`): bei „ja" Hinweis auf die manuellen SonarCloud-Schritte +
  Deeplink Anhang AA; BOO-122-Warnung-Referenz auf „Anhang AA" präzisiert.
- **Provider-Postflight** (`provider-postflight.md` + `.en.md`): Callout, dass die SonarCloud-seitige
  Prüfung hier stattfindet, mit Deeplink Anhang AA.

## Acceptance Criteria
1. HANDBUCH-Runbook (DE+EN, stabiler Anker) mit zwei Szenarien (A: Account existiert / B: von 0). ✓
2. Hinweis: erster `git push` triggert `sonar.yml`. ✓ (Anhang AA + D.5)
3. Bootstrap D.5 weist bei „ja" auf SonarCloud-Schritte + Deeplink hin. ✓
4. Provider-Postflight (BOO-58) verlinkt auf das Runbook. ✓

## Anti-Fabrikation
Story-Befund nannte HANDBUCH-Sonar-Verdrahtung „Z.1418" u.a. — die Zeilen sind gedriftet, aber Substanz
(SonarLint Connected Mode, Required-Check `SonarCloud`) bestätigt. A–Z-Anhänge waren erschöpft → neuer
**Anhang AA** als stabiler Anker (statt SonarCloud unter „Kunden-Onboarding Z" zu quetschen).

## Result
Umgesetzt im Branch `feat/boo-docs-secondbrain`. Schließt den BOO-122-Forward-Verweis (Anhang AA). DE+EN.
