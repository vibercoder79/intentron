# INTENTRON v0.5.0 ‚Äî Release Overview

Status: 2026-05-31 ¬∑ Previous tag: `v0.4.0` (Maintainability, Waves Y-Z)

This version brings a targeted building block from the competitive analysis (Matt Pocock's `skills` vs. BMAD vs. INTENTRON): the **Ubiquitous Language `CONTEXT.md`** ‚Äî the one point from the lightweight camp that meaningfully docks onto INTENTRON, because the vocabulary in regulated domains is **legally charged**.

## The Big Picture

| Wave | Story | What improved | Status |
|-------|-------|--------------------------|--------|

## Wave AA ‚Äî CONTEXT.md Ubiquitous Language (BOO-91)

**Problem before:** Without a fixed vocabulary, the AI invents synonyms (`User` / `Customer` / `Betroffener` for the same entity) ‚Üí fragmented code, poor `grep`-ability, misalignment, token waste. For a regulated target audience, the vocabulary is additionally legally defined (`Betroffener` = GDPR Art. 4; nDSG says `Bearbeitung` instead of `Verarbeitung`).

**What is there now:**

- **Pre-filled framework base** `bootstrap/references/context-base.md` (DE+EN): a table `kanonisch | verboten | quelle`. Compliance vocabulary bound to GDPR/nDSG, governance vocabulary bound to INTENTRON ‚Äî **every term with a source** (audit evidence).
- **Project overlay** `CONTEXT.md` in the project root: seeded from the base during `/bootstrap` plus a `## Projekt-Domaene` section that the operator fills in. The overlay **survives framework updates** (migration never overwrites it).
- **Default = guidance, not a hard gate:** the AI reads `CONTEXT.md` while writing and adheres to the canonical vocabulary ‚Äî it is guided, not blocked.
- Wiring: Bootstrap phase 4.3b + `CLAUDE.md` reference, `CONVENTIONS.md` entry, HANDBUCH **Anhang X**, `migrate_boo_91` (idempotent), bootstrap 3.34.0 ‚Üí 3.35.0.

**Effect:** Terms are consistent and ‚Äî in the compliance part ‚Äî bound to the legal basis. This sharpens the audit trail (nDSG `Bearbeitung` vs. GDPR `Verarbeitung`) and reduces misalignment, without friction from a hard gate.

**Design decision:** The pattern originates from Pocock's `skills` repo ‚Äî **rebuilt, no code taken over**. Deliberately only the **guidance layer**; the **enforcement** (dpo control "vocabulary follows CONTEXT.md" + Layer-0 bodyguard warning in PII paths) is spun off as a later, opt-in expansion stage ‚Äî so that nothing tips toward heavyweight.
