## Wave AS — Complementary tooling: setup-checklist (BOO-113)

The `setup-checklist` (best-practice machine setup for Claude Code) is provided decoupled from the framework as its own **public** repo — https://github.com/vibercoder79/claude-code-setup-checklist (MIT, DE+EN). Reason: intentron is public, the skill repo is private; only a public standalone repo can be linked from the public README. intentron points to it in the README section "Complementary tooling — machine setup" and in HANDBUCH appendix Z.4, including the order `setup-checklist global → /bootstrap → setup-checklist audit/additive` and the rule of thumb "machine + hygiene = the checklist · project governance = bootstrap". The sync (outside this repo) runs via an extended `publish_skill.py` (fan-out to claudecodeskills + the public repo); the single source of truth stays the local skill folder.
