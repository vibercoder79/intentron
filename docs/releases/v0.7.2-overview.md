# 🇬🇧 English Version

# INTENTRON v0.7.2 — Release Overview

As of: 2026-06-01 · Predecessor tag: `v0.7.1` (bootstrap preparation checklist)

> Patch release: turns the EU AI Act add-on (v0.7.0) into a true conditional opt-in. Detail spec: `specs/BOO-105.md`.

## The Big Picture

| Wave | Story | What improved | Status |
|------|-------|---------------|--------|
| **AK** | BOO-105 | **EU AI Act true opt-in** — catalogue removed from auto-load (`controls/optional/`), executable bootstrap phase 4.4n-bis | ✅ done |

## Wave AK — EU AI Act conditional opt-in (BOO-105)

**Problem:** In v0.7.0 the EU AI Act catalogue effectively ran on every dpo audit (the runner loads all `controls/*.yml`), even in privacy projects without an AI component → noise; and there was no executable bootstrap phase, only a note.

**Fix:** moved the catalogue to `dpo/controls/optional/eu-ai-act.yml` (the non-recursive framework glob no longer auto-loads it); new bootstrap phase **4.4n-bis** copies it into the project overlay `.claude/dpo/controls/` on `[x] EU AI Act` and renders `AI_SYSTEM.md`. Docs (A.4, dpo/SKILL, README, catalogue header) aligned, DE+EN.

**Effect:** end-to-end deterministic, guaranteed active only for AI projects. No new runner code.
