# INTENTRON v0.4.0 — Release-Ueberblick

Stand: 2026-05-31 · Vorgaenger-Tag: `v0.3.0` (Security-Wave V-X)

Diese Version bringt zwei **Wartbarkeits-Wellen** (Y, Z), die direkt aus der Security-Wave entstanden sind: Beim coverage-check-Bugfix (BOO-88) wurde sichtbar, dass gescaffoldete Hooks dreifach gepflegt werden und dass Feld-Fixes keinen Rueckweg in die Quelle haben. v0.4.0 schliesst beide Luecken — das Framework wird besser **wartbar** und kann Feld-Erkenntnisse **zurueckfliessen** lassen.

> Detail-Notes: `wave-y-hook-single-source.md`, `wave-z-contribute-back.md` (im Ordner `docs/releases/`).

## Das grosse Bild

| Welle | Story | Was sich verbessert hat | Status |
|-------|-------|--------------------------|--------|
| **Y** | BOO-89 | **Single-Source** fuer `coverage-check.sh` (statt dreifach) + **Drift-Guard** in CI | ✅ done |
| **Z** | BOO-90 | **Contribute-Back-Schleife** — Feld-Fixes erzeugen Patch + Issue-Vorschlag fuer die Quelle | ✅ done |

Roter Faden: **Das Framework pflegt sich selbst sauberer.** Beide Wellen sind dependency-frei (bash + python3-Stdlib), getestet und veraendern das Laufzeitverhalten der Hooks nicht.

## Wave Y — coverage-check Single-Source + Drift-Guard (BOO-89)

**Problem vorher:** `coverage-check.sh` lag dreifach im Repo (DE-Template, EN-Template, eingebetteter Heredoc in `migrate-to-v2.sh`). Der BOO-88-Fix brauchte drei synchrone Edits — Drift war absehbar.

**Was jetzt da ist:**

- **Kanonische Quelle** `bootstrap/references/hooks/coverage-check.sh` (v2, inkl. BOO-88-Fix) — eine Datei statt drei.
- `migrate_boo_15` **kopiert** aus der Quelle (kein `COVCHECK_EOF`-Heredoc mehr); die Templates zeigen per Pointer darauf. Bootstrap rendert verbatim aus der kanonischen Datei.
- **Drift-Guard** `bootstrap/scripts/check-hook-sources.sh` (lokal **und** in CI via `.github/workflows/hook-sources.yml`): prueft kanonische Validitaet, Nicht-Doppelpflege und Byte-Identitaet eines frisch migrierten Hooks. Restliche eingebettete Hooks werden als Folge-Kandidaten gelistet.

**Effekt:** Ein Hook-Fix passiert kuenftig **einmal**; CI faengt Drift ab. bootstrap 3.33.0 → 3.34.0.

## Wave Z — Contribute-Back-Schleife (BOO-90)

**Problem vorher:** Wenn ein deployter Factory-Nutzer einen Bug in einem kopierten Governance-Artefakt lokal fixt (wie coverage-check aus privacy-proxy / PP-001), gab es keinen systematischen Rueckweg in die Quelle — der Rueckfluss lief manuell.

**Was jetzt da ist:**

- **`bootstrap/scripts/contribute-fix.sh`** (Feld→Quelle): erkennt lokal geaenderte Framework-Artefakte (zunaechst die gescaffoldeten Hooks) und erzeugt pro Abweichung einen **Patch** + einen **Issue-Vorschlag** unter `contribute-back/`.
- **Kein Auto-Push/Auto-PR** — der Operator prueft und reicht selbst ein (kein ungefragtes Abfliessen aus Kundenprojekten).
- **Kopplung an BOO-89:** je mehr Hooks Single-Source werden, desto mehr deckt `contribute-fix` automatisch ab.
- HANDBUCH **Anhang W**.

**Effekt:** Die beiden Richtungen der Schleife sind jetzt da — Quelle→Feld (Versions-Marker + ersetzende Migration, BOO-88) und Feld→Quelle (BOO-90).

## Folge-Kandidaten

- Weitere gescaffoldete Hooks (spec-gate, dependency-check, pre-edit-bodyguard, …) auf Single-Source umstellen (Drift-Guard listet sie).
- `contribute-fix` auf weitere Artefakt-Typen generalisieren + optionaler `/contribute-fix`-Skill-Wrapper.
