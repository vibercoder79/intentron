Stand: 2026-05-31 · Vorgaenger-Tag: `v0.3.0` (Security-Wave V-X)

Diese Version bringt zwei **Wartbarkeits-Wellen** (Y, Z), die direkt aus der Security-Wave entstanden sind: Beim coverage-check-Bugfix (BOO-88) wurde sichtbar, dass gescaffoldete Hooks dreifach gepflegt werden und dass Feld-Fixes keinen Rueckweg in die Quelle haben. v0.4.0 schliesst beide Luecken — das Framework wird besser **wartbar** und kann Feld-Erkenntnisse **zurueckfliessen** lassen.

> Detail-Notes: `wave-y-hook-single-source.md`, `wave-z-contribute-back.md` (im Ordner `docs/releases/`).

| Welle | Story | Was sich verbessert hat | Status |

| **Z** | BOO-90 | **Contribute-Back-Schleife** — Feld-Fixes erzeugen Patch + Issue-Vorschlag fuer die Quelle | ✅ done |

## Wave Y — coverage-check Single-Source + Drift-Guard (BOO-89)

**Problem vorher:** `coverage-check.sh` lag dreifach im Repo (DE-Template, EN-Template, eingebetteter Heredoc in `migrate-to-v2.sh`). Der BOO-88-Fix brauchte drei synchrone Edits — Drift war absehbar.

**Was jetzt da ist:**

- **Kanonische Quelle** `bootstrap/references/hooks/coverage-check.sh` (v2, inkl. BOO-88-Fix) — eine Datei statt drei.
- `migrate_boo_15` **kopiert** aus der Quelle (kein `COVCHECK_EOF`-Heredoc mehr); die Templates zeigen per Pointer darauf. Bootstrap rendert verbatim aus der kanonischen Datei.
- **Drift-Guard** `bootstrap/scripts/check-hook-sources.sh` (lokal **und** in CI via `.github/workflows/hook-sources.yml`): prueft kanonische Validitaet, Nicht-Doppelpflege und Byte-Identitaet eines frisch migrierten Hooks. Restliche eingebettete Hooks werden als Folge-Kandidaten gelistet.

**Effekt:** Ein Hook-Fix passiert kuenftig **einmal**; CI faengt Drift ab. bootstrap 3.33.0 → 3.34.0.

## Wave Z — Contribute-Back-Schleife (BOO-90)

**Problem vorher:** Wenn ein deployter Factory-Nutzer einen Bug in einem kopierten Governance-Artefakt lokal fixt (wie coverage-check aus privacy-proxy / PP-001), gab es keinen systematischen Rueckweg in die Quelle — der Rueckfluss lief manuell.

**Was jetzt da ist:**

- **`bootstrap/scripts/contribute-fix.sh`** (Feld→Quelle): erkennt lokal geaenderte Framework-Artefakte (zunaechst die gescaffoldeten Hooks) und erzeugt pro Abweichung einen **Patch** + einen **Issue-Vorschlag** unter `contribute-back/`.
- **Kein Auto-Push/Auto-PR** — der Operator prueft und reicht selbst ein (kein ungefragtes Abfliessen aus Kundenprojekten).
- **Kopplung an BOO-89:** je mehr Hooks Single-Source werden, desto mehr deckt `contribute-fix` automatisch ab.
- HANDBUCH **Anhang W**.

**Effekt:** Die beiden Richtungen der Schleife sind jetzt da — Quelle→Feld (Versions-Marker + ersetzende Migration, BOO-88) und Feld→Quelle (BOO-90).

## Folge-Kandidaten

- Weitere gescaffoldete Hooks (spec-gate, dependency-check, pre-edit-bodyguard, …) auf Single-Source umstellen (Drift-Guard listet sie).
- `contribute-fix` auf weitere Artefakt-Typen generalisieren + optionaler `/contribute-fix`-Skill-Wrapper.

# 🇬🇧 English Version

# INTENTRON v0.4.0 — Release Overview

Status: 2026-05-31 · Predecessor tag: `v0.3.0` (Security Wave V-X)

This version brings two **maintainability waves** (Y, Z) that emerged directly from the Security Wave: During the coverage-check bugfix (BOO-88), it became apparent that scaffolded hooks are maintained in triplicate and that field fixes have no path back into the source. v0.4.0 closes both gaps — the framework becomes more **maintainable** and can let field insights **flow back**.

> Detail notes: `wave-y-hook-single-source.md`, `wave-z-contribute-back.md` (in the `docs/releases/` folder).

## The Big Picture

| Wave | Story | What improved | Status |
|-------|-------|--------------------------|--------|
| **Y** | BOO-89 | **Single source** for `coverage-check.sh` (instead of triplicate) + **drift guard** in CI | ✅ done |
| **Z** | BOO-90 | **Contribute-back loop** — field fixes generate a patch + issue proposal for the source | ✅ done |

Common thread: **The framework keeps itself cleaner on its own.** Both waves are dependency-free (bash + python3 stdlib), tested, and do not change the runtime behavior of the hooks.

---

## Wave Y — coverage-check Single-Source + Drift Guard (BOO-89)

**Problem before:** `coverage-check.sh` existed in triplicate in the repo (DE template, EN template, embedded heredoc in `migrate-to-v2.sh`). The BOO-88 fix required three synchronous edits — drift was foreseeable.

**What is there now:**

- **Canonical source** `bootstrap/references/hooks/coverage-check.sh` (v2, including the BOO-88 fix) — one file instead of three.
- `migrate_boo_15` **copies** from the source (no more `COVCHECK_EOF` heredoc); the templates point to it via a pointer. Bootstrap renders verbatim from the canonical file.
- **Drift guard** `bootstrap/scripts/check-hook-sources.sh` (locally **and** in CI via `.github/workflows/hook-sources.yml`): checks canonical validity, no double maintenance, and byte identity of a freshly migrated hook. The remaining embedded hooks are listed as follow-up candidates.

**Effect:** A hook fix now happens **once**; CI catches drift. bootstrap 3.33.0 → 3.34.0.

## Wave Z — Contribute-Back Loop (BOO-90)

**Problem before:** When a deployed factory user locally fixes a bug in a copied governance artifact (like coverage-check from privacy-proxy / PP-001), there was no systematic path back into the source — the flow back ran manually.

**What is there now:**

- **`bootstrap/scripts/contribute-fix.sh`** (field→source): detects locally modified framework artifacts (initially the scaffolded hooks) and generates per deviation a **patch** + an **issue proposal** under `contribute-back/`.
- **No auto-push/auto-PR** — the operator reviews and submits it themselves (no unsolicited outflow from customer projects).
- **Coupling to BOO-89:** the more hooks become single-source, the more `contribute-fix` automatically covers.
- HANDBUCH **Appendix W**.

**Effect:** Both directions of the loop are now in place — source→field (version marker + replacing migration, BOO-88) and field→source (BOO-90).

---

## Follow-up Candidates

- Convert further scaffolded hooks (spec-gate, dependency-check, pre-edit-bodyguard, …) to single-source (the drift guard lists them).
- Generalize `contribute-fix` to further artifact types + optional `/contribute-fix` skill wrapper.
