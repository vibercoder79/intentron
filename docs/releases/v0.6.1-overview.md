## Wave AF — Onboarding-Fix: Install + Quickstart + Self-Install/Self-Update (BOO-96)

**Problem vorher:** Der dokumentierte Installations-Befehl war nach dem Rebrand (code-crash-framework → `intentron`) kaputt: HANDBUCH §4 Schritt 3 (DE+EN) klonte `vibercoder79/claudecodeskills` und nutzte den Pfad `intentron/bootstrap` — den es dort nicht gibt (404). Der Bootstrap-Skill liegt im public Repo `vibercoder79/intentron` direkt im Root. Ein neuer Nutzer bekam „No such file or directory". Die README zeigte den Install-Schritt gar nicht, nur `/bootstrap` (das ohne Installation ins Leere geht).

**Was jetzt da ist:**

- **Install-Befehl korrigiert** (HANDBUCH.md + HANDBUCH.en.md, je Schritt 3 + Update-Anhang; bootstrap/README): Clone auf `https://github.com/vibercoder79/intentron.git` (public), `git sparse-checkout set bootstrap`, `cp -r bootstrap ~/.claude/skills/`.
- **README-Quickstart** (DE „Schnellstart" + EN „Quickstart") mit drei Einstiegswegen: **A** manuell + `/bootstrap`, **B** AI-Self-Install-Prompt (KI klont selbst + liest SKILL.md), **C** AI-Self-Update-Prompt für alte/Brownfield-Installationen — sicherer gestufter Flow nach `bootstrap/references/framework-upgrade.md` (inspect → apply-safe → apply-with-confirmation; nutzt `migrate-to-v2.sh` + `verify-setup.sh`, überschreibt lokale Entscheidungen/Secrets nie blind).

**Effekt:** Der Einstieg funktioniert wieder für frische Nutzer, ist sichtbar, und es gibt jetzt drei Wege — Hand-Install, KI-Selbst-Install und KI-Selbst-Update einer Altinstallation. Reine Doku.

## Wave AF — Onboarding fix: install + quickstart + self-install/self-update (BOO-96)

**Problem before:** After the rebrand (code-crash-framework → `intentron`) the documented install command was broken: HANDBUCH §4 step 3 (DE+EN) cloned `vibercoder79/claudecodeskills` and used the path `intentron/bootstrap` — which does not exist there (404). The bootstrap skill lives in the public repo `vibercoder79/intentron` at the repo root. A new user got "No such file or directory". The README did not show the install step at all, only `/bootstrap` (which goes nowhere without an install).

**What is there now:**

- **Install command corrected** (HANDBUCH.md + HANDBUCH.en.md, step 3 + update appendix each; bootstrap/README): clone `https://github.com/vibercoder79/intentron.git` (public), `git sparse-checkout set bootstrap`, `cp -r bootstrap ~/.claude/skills/`.
- **README quickstart** (DE "Schnellstart" + EN "Quickstart") with three entry paths: **A** manual + `/bootstrap`, **B** AI self-install prompt (the tool clones itself + reads SKILL.md), **C** AI self-update prompt for old/brownfield installs — safe staged flow per `bootstrap/references/framework-upgrade.md` (inspect → apply-safe → apply-with-confirmation; uses `migrate-to-v2.sh` + `verify-setup.sh`, never overwrites local decisions/secrets blindly).

**Effect:** Onboarding works again for fresh users, is visible, and there are now three paths — hand-install, AI self-install, and AI self-update of an old install. Docs only.
